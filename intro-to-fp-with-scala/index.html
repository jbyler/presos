What is functional programming?

  • Programming with functions.
  • A function f: A => B relates every value of type A to exactly one value of type B

Referential transparency

  • every occurrence of an expression can be replaced with its value without affecting the result
  • f is pure if f(x) is referentially transparent for all referentially transparent choices of x
  • square: Int => Int = x => x * x is a function
  • later: Long => Long = x => System.currentTimeMillis + x is NOT a function

A function cannot have side-effects

  • Assigning to variables
  • Mutating data structures
  • Throwing exceptions
  • Reading/writing files
  • etc

A functional program...

  • is NOT a sequence of instructions
  • is a single referentially transparent expression
  • describes what, not how

So what is functional programming?

A style of writing code with some constraints
  • only referential expressions
  • only pure functions
  • no side-effects, ever

Why FP?

  • modularity
  • composability
  • easier to reason about -> fewer bugs
  • scale indefinitely
